Canyon Ranch Database Project

Client: Canyon Ranch Wellness Resorts

Overview: Designed and deployed a centralized relational and dimensional data system for Canyon Ranch to replace fragmented legacy software. The goal was to improve operational coordination, reservation handling, and department-level reporting.

Project Objectives

  • Eliminate reconciliation across multiple disconnected systems
  • Enable referral tracking from website, affiliates, and guests
  • Capture feedback tied to specific services and staff
  • Provide reporting visibility by department and time period

Solution Design

Operational Database (OLTP): A fully normalized SQL Server schema was built to support transactional integrity. It covered guests, reservations, services, staff, rooms, referrals, and feedback.

Data Warehouse (OLAP): A snowflake schema supported trend analysis and reporting across service categories, booking channels, and satisfaction scores. Dimensions included date, customer, service, and service type.

ETL/ELT Pipelines: Structured loading and in-database transformations computed booking volume, average ratings, and usage metrics directly within the warehouse model.

Key Insights

  • Referral Source ROI: Affiliate referrals produced the highest revenue per guest, though least used. Website referrals brought volume but lower spend per booking.
  • Service Trends: Spa services dominated bookings, with Q2 and weekends showing spikes. Fitness and coaching showed room for growth during off-peak times.
  • Feedback Analytics: Spa services scored highest in satisfaction, while nutrition/coaching feedback varied by staff — highlighting staff-level training needs.
  • Department Visibility: The warehouse enabled real-time trend slicing by service type, guest segment, and referral source, supporting targeted marketing and scheduling decisions.

Assumptions & Caveats

  • All data used was synthetically generated to reflect realistic logic
  • Modeled sensitive data types, assuming consent and regulatory compliance (HIPAA)
  • Design assumes security features (encryption, access controls) would be implemented in production

Tech Stack

  • MS SQL Server
  • Cursor IDE
  • ERDPlus
  • Markdown / Word (docs)
  • GPT-4 (sample data generation)
